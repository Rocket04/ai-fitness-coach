# CONVENTIONS.md — Свод правил и ограничений для разработки

## ?? Главная цель
Ты — Senior React-разработчик, помогающий создать offline-first фитнес-приложение. Все решения должны приниматься через призму главной философии продукта: "Открыл утром — увидел, что делать. Никаких выборов".

## ?? Использование AI-модели (Ring-2.6-1T)
- **Модель:** `inclusionai/ring-2.6-1t:free` через OpenRouter.
- **Режим мышления (reasoning effort):**
  - `high` — для обычных задач (доработка компонентов, исправления, рефакторинг).
  - `xhigh` — для сложных архитектурных решений (проектирование структуры БД, алгоритмы адаптации, core/engine.js).
  - В промптах можно указывать желаемый режим явно: `[reasoning: xhigh]`.
- **Формат ответа:** При необходимости можно запрашивать строгий JSON (response_format), но по умолчанию — свободный текст с кодом.
- **OpenRouter:** Для доступа используй переменные окружения `OPENROUTER_API_KEY` и `AIDER_MODEL=openrouter/inclusionai/ring-2.6-1t:free`.

## ?? Технический стек (строго)
- **React 18.2.0** и **ReactDOM 18.2.0** (CDN, фиксированные версии)
- **Dexie.js 4.x** (CDN) для IndexedDB
- **Workbox 7.x** (CDN) для Service Worker
- **Babel** использовать **локально** для компиляции JSX. В браузер должен попадать уже чистый JavaScript (ES6 модули).
- **Никаких других библиотек** без явного согласования.
- **Никакого Node.js** на бэкенде. Приложение — полностью статическое.
- **Никаких внешних API** или сетевых запросов, кроме CDN.

## ?? Стиль кода
- **Только ES6 модули** (`export`/`import`).
- **Функциональные компоненты** и хуки React (`useState`, `useEffect`, `useMemo`, `useRef`).
- `const` и `let`, **никакого `var`**.
- Компоненты в `PascalCase` (`Pill.js`, `TodayPage.js`), утилиты в `camelCase` (`storage.js`, `engine.js`).
- Комментарии на русском, краткие, только для объяснения сложной логики.
- Максимальная длина функции — 30 строк. Файла — 200 строк.

## ??? Структура и архитектура
- Строго придерживаться файловой структуры, описанной в `PROJECT_CONTEXT.md`.
- **`config/constants.js`**: все статические данные (зоны, упражнения, планы тренировок).
- **`core/engine.js`**: вся бизнес-логика (подсчёт Recovery Score, адаптация, APRE, советы).
- **`core/storage.js`**: все операции с IndexedDB через Dexie.js.
- **`ui/components/`**: переиспользуемые UI-кирпичики.
- **`ui/pages/`**: компоненты-страницы для каждой вкладки.
- **Code Splitting**: каждая страница загружается лениво через `React.lazy(() => import(...))`.

## ?? Дизайн и UX (обязательно)
- **Mobile-first**: `max-width: 500px`, все интерактивные элементы не менее 44x44px.
- Тёмная тема в соответствии с дизайн-токенами.
- Кнопки: зелёная (`btn-green`) — выполнить, серая (`btn`) — сохранить, красная (`btn-red`) — сброс.
- Pill-индикаторы: зелёный/жёлтый/красный для статуса дня.

## ?? Категорически запрещено
- **Использовать jQuery, Bootstrap** и любые CSS-фреймворки.
- **Хранить большие данные в LocalStorage** (только настройки). Все данные — в IndexedDB.
- **Делать запросы к внешним API.**
- **Удалять или радикально менять существующую логику** (Recovery Score, APRE) без явного запроса.
- **Использовать Babel Standalone в production.** Код должен быть предварительно скомпилирован.
- **Писать неадаптивные элементы** (фиксированная ширина, отсутствие `viewport-fit=cover`).
- **Добавлять анимации дольше 150мс** без необходимости.
