# Session: Comprehensive UI Audit

**Date:** 2026-05-30
**Goal:** Systematically audit ALL UI components for CSS module issues, missing global styles, i18n gaps, data-testid mismatches, and visual consistency bugs.

## Issues Found

### Phase 1: CSS Module Issues
- **ExerciseCard.module.css**: Dead classes `.exercise-list` and `.exercise-sets` — defined but never referenced via `styles[]` (app uses global equivalents from styles.css)
- **ScaleSelector.module.css**: `--color-text-tertiary` CSS var doesn't exist in design tokens (line 12)

### Phase 2: Global Style Issues

- `set-progress-bar` class used in WorkoutMode.jsx line 223 but NOT defined in styles.css
- `stat-trend` class used in StatBox.jsx line 19 but NOT defined in styles.css

### Phase 3: i18n Issues
- Multiple hardcoded Russian strings throughout pages (TodayPage, ProfilePage, CheckinForm, LogPage, etc.)
- `t('training.pullUps')`, `t('training.pushUps')`, `t('training.plank')` referenced in WorkoutMode.jsx — confirmed in ru.json but not in en.json (they're under `units.training` in en.json instead)
- `t('score.above')` etc. used in TodayPage — confirmed key exists at top level in both locales

### Phase 4: data-testid Issues
- 50+ selectors in e2e/utils/selectors.ts have no corresponding data-testid in UI components
- Selectors file itself acknowledges this with a NOTE comment

### Phase 5: Visual Issues
- Missing hover/active/focus states on plain `<button>` elements without `.btn` class
- `workout-mode-overlay` has no CSS class defined — relies entirely on inline styles
