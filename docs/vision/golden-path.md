# 🏔️ Smart Fitness Coach — The Golden Path (User Experience Vision)

## Акт I: Знакомство и Доверие (День 0)

### Сцена 1: Первый Запуск

Атлет Марк впервые открывает приложение. Он не видит пустой дашборд, требующий данных. Вместо этого его встречает тёплый, но предельно лаконичный онбординг, который говорит с ним о его целях.

1. **Экран Ценности ("Зачем"):** Не "Добро пожаловать в приложение", а **"Тренировки, которые слушают твоё тело"**. Одна фраза, которая обещает решить его главную боль — постоянные мысли "готов ли я сегодня?"
2. **Выбор пути ("Что"):** *"К чему ты стремишься?"* — карточки: Сила, Выносливость, Реабилитация, Гибкость. Марк выбирает "Сила". Приложение автоматом подбирает протокол APRE_6 как основной.
3. **Устройства ("Чем"):** *"Как будем измерять восстановление?"* — HRV-монтор, Умные часы, Вручную. Марк выбирает "Вручную". Приложение не упрекает его, а говорит: "Отлично, будем опираться на твои ощущения. Сон и пульс сможем загрузить потом".
4. **Первое обязательство:** Марк нажимает **"Начать тренироваться"**. Никаких 7 шагов, всё уместилось в 3 экрана. Приложение создаёт его первый, адаптированный план на завтра.

### Сцена 2: Вторая встреча (День 1, Утро)

Марк открывает приложение утром. Холодный старт.

1. **Герой-экран:** В центре — большое кольцо с надписью **"Заполни чек-ин"**. Это не 0 баллов, а призыв к действию.
2. **Чек-ин (Экономия касаний):** Марк вводит сон (7ч) и пульс (65). Всё. Это 15 секунд. **Моментальная магия:** Кольцо окрашивается в зелёный (Recovery Score 82). Появляется текст: **"Готов. Сегодня тяжёлая тренировка."**
3. **План (Без шума):** Марк листает ниже и видит: "Тренировка A: Верх тела". Карточки упражнений уже раскрыты. Он видит подтягивания (3х6), жим лёжа (3х8). Всё чётко, без лишней теории.

## Акт II: Основная Петля (День 1, Вечер)

### Сцена 3: Тренировка

Марк в зале. Открывает приложение.

1. **Режим "Тренировка":** Большая кнопка **"Начать"**. На весь экран — только текущее упражнение и таймер отдыха.
2. **Трекинг (Тактильный):** *Подтягивания:* Марк делает 6 раз. Интерфейс показывает строку "Подход 1 [✓] Повторения [6]". Марк свайпает вправо по строке подхода или тыкает в большой чекбокс — подход записан. Тут же высвечивается таймер отдыха 120с. *Экран не гаснет.* Марк не разблокирует телефон между подходами.
3. **APRE на лету:** Последний подход (AMRAP). Марк делает 8 раз вместо запланированных 6. Приложение **сразу** показывает: "📈 +2.5 кг к жиму на следующей неделе".
4. **Завершение:** Все подходы отмечены. Марк нажимает **"Завершить"**. Его спрашивают: "Как ощущения? (RPE)". Он выбирает 7 из 10. Всё. Тренировка записана.

## Акт III: Петля Обратной Связи (День 8)

### Сцена 4: Адаптация

Марк тренируется неделю. Он выполнял план на 85%.

1. **Понедельник. Утро.** Марк открывает приложение.
2. **Данные уже там:** Он ещё вчера загрузил CSV из Health Sync. Сон и пульс за всю неделю подтянулись автоматом.
3. **Адаптивный план:** На карточке тренировки он видит плашку: **"🔼 Прогресс! Объём увеличен на 20% — отличная работа на прошлой неделе"**. Вместо 3х6 подтягиваний теперь стоит 4х6. Система доверяет ему больше.
4. **Предиктив:** Внизу экрана появляется совет: "Твой пульс покоя снизился на 2 уд/мин за неделю. Сердечно-сосудистая форма улучшается. Продолжай в том же духе."

### Сцена 5: Что-то идёт не так

Марк плохо спит 2 дня. Его Recovery Score падает до 43 (жёлтый).

1. **Утро.** Кольцо горит жёлтым. Заголовок: **"Будь осторожен. Восстановление ниже среднего."**
2. **План:** Система автоматом меняет режим. "Тяжёлая тренировка" заменена на "Техническая". Жим лёжа 3х8 с весом 90% заменён на 3х5 с весом 70% + работа над техникой. Плашка: **"⚠️ Вес снижен для защиты ЦНС"**.
3. **Честность:** Марк нажимает "Почему?" и видит: "HRV упал на 12% за 2 дня, пульс покоя вырос на 4 уд/мин. Вероятная причина: недосып. Рекомендация: сегодня лёгкая нагрузка, завтра попробуем снова."

### E2E Test Checklist (based on this vision)

1. **Onboarding Flow:** User sees the value screen, selects a goal and input type, and arrives at the main plan screen.
2. **Daily Check-in:** The recovery ring updates instantly with a color and score upon logging sleep and heart rate.
3. **Workout Tracking:** Users can mark sets as complete, see a progress bar, enter RPE, and save the workout.
4. **Weekly Adaptation:** An adherence banner announces volume changes, and subsequent workouts reflect the updated plan.
5. **Low Recovery Handling:** Poor check-in data triggers a yellow/red readiness status, reduces workout intensity, and provides an honest "Why?" explanation.
6. **CSV Import:** Uploading a Health Sync file successfully merges data without overwriting manual entries.
7. **Navigation:** Bottom tabs switch between the main app views seamlessly.
