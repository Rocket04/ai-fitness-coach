# 🏋️‍♂️ Стратегия разработки фитнес-трекера с помощью AI-стека

Этот документ определяет роли и сценарии использования бесплатных нейросетей в качестве **Rubber Ducking (резиновой уточки)**, архитекторов и оркестраторов идей для проекта фитнес-трекера и генератора планов тренировок.

---

## 🗺️ Карта распределения ролей и лимитов


| Нейросеть и модель | Лимиты в веб-интерфейсе | Идеальная роль в проекте | Основные задачи |
| :--- | :--- | :--- | :--- |
| 🐉 **DeepSeek V4 Pro** | **Без лимитов** | **Основная «Уточка» (Rubber Duck)** | Написание рутинного кода функций, калькуляторов, багфиксинг часами. |
| 🦩 **Claude 4.6 Sonnet** | **Очень жесткие** (5-7 сообщений) | **Технический директор (Tech Lead)** | Проектирование архитектуры БД, нарезка задач, аудит сложной логики. |
| 🪐 **Gemini 3.5 Flash** | **Средние** (Падает скорость) | **Аналитик базы знаний** | Переваривание больших ТЗ, гайдлайнов API (Apple Health) и книг по фитнесу. |
| 🧠 **ChatGPT (GPT-5)** | **Гибкие / Щедрые** | **Продуктолог и UI/UX дизайнер** | Проектирование экранов приложения, генерация референсов, написание текстов. |

## 🛠️ Инструкция: Как использовать каждую модель на практике

### 1. 🐉 DeepSeek V4 Pro — Повседневный Кодинг и Обсуждение
* **Когда открывать**: Когда нужно сесть и писать код часами.
* **Как промптить**: 
  > *"Я пишу фитнес-трекер. Вот структура моей базы данных: [вставить структуру]. Напиши на [ваш язык] функцию, которая рассчитывает дневную норму калорий по формуле Миффлина-Сан Жеора и сохраняет результат."*
* **Фишка**: Обсуждайте с ним ошибки компиляции, он не отключит вас из-за лимитов посреди важной сессии отладки.

### 2. 🦩 Claude 4.6 Sonnet — Контроль Качества и Архитектура
* **Когда открывать**: 1-2 раза за сессию разработки, когда нужно принять важное решение.
* **Как промптить**:
  > *"Посмотри на эту схему связей между тренировочными планами и днями календаря: [код]. Есть ли здесь архитектурная ошибка, которая помешает масштабированию, если пользователь пропустит тренировку? Предложи идеальное решение."*
* **Фишка**: Берегите сообщения. Сюда отправляется только код, прошедший первую сборку у DeepSeek.

### 3. 🪐 Gemini 3.5 Flash — Работа с Огромным Контекстом
* **Когда открывать**: На старте фичи, когда нужно изучить документацию.
* **Как промптить**:
  > *[Загрузить PDF-файл документации Apple HealthKit или книгу по спортивной физиологии]* 
  > *"Опираясь строго на этот документ, составь пошаговый алгоритм: как рассчитывать зоны пульса для жиросжигания у человека 30 лет. Выдай чистую логику, без кода."*
* **Фишка**: Используйте окно в 1 млн токенов, чтобы скармливать модели целые спортивные справочники.

### 4. 🧠 ChatGPT — Интерфейс и Логика Продукта
* **Когда открывать**: Когда нужно придумать, как приложение будет выглядеть и общаться с пользователем.
* **Как промптить**:
  > *"Я проектирую экран создания плана тренировок. Опиши лучшие UX-практики: сколько шагов должно быть, какие элементы управления (селекторы, инпуты) использовать, чтобы пользователю было удобно вводить веса на тренировке."*
